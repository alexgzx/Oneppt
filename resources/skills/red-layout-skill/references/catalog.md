# Red-note Layout Catalog

This catalog describes 1242x1660 Xiaohongshu/red-note structures. It is structural only: style controls color, type mood, decoration, shadows, and texture.

## Zone Skeletons

### `hook-body-takeaway`

- **Use for**: most red-note pages.
- **Zone sketch**: large hook, useful body sections, compact bottom takeaway/source.
- **Balance rule**: each zone has a job: stop, teach, remember.
- **Failure sign**: the page has useful content but no stop reason or final memory point.

### `saveable-list`

- **Use for**: checklists, tips, pitfalls, templates, step guides.
- **Zone sketch**: hook, 3-6 rows/cards, bottom action or summary.
- **Balance rule**: rows must scan quickly and feel collectible.
- **Failure sign**: rows become paragraphs or unrelated cards.

### `single-value-object`

- **Use for**: one data point, framework, template, or comparison.
- **Zone sketch**: hook, one dominant value object, short explanation/takeaway.
- **Balance rule**: the value object should be the reason to save the page.
- **Failure sign**: the main object is buried in surrounding explanation.

## Patterns

### `cover-hook`

- **Input shape**: title hook, one benefit/claim, optional context.
- **Structure recipe**: large headline, short subtitle/context, one visual anchor or proof chip.
- **Budget rule**: hook owns the top/center; support is minimal.
- **Failure signs**: no immediate stop reason; metadata competes with the hook.

### `saveable-checklist`

- **Input shape**: 3-6 practical tips, pitfalls, takeaways, or rules.
- **Structure recipe**: labeled rows/cards with short titles and one-line explanations.
- **Budget rule**: rows fill the value body; bottom summarizes the memory point.
- **Failure signs**: item text becomes long; list lacks grouping or final takeaway.

### `step-guide`

- **Input shape**: 3-5 ordered actions, stages, or methods.
- **Structure recipe**: numbered vertical sequence with action label + concise detail.
- **Budget rule**: sequence uses the middle height with consistent row rhythm.
- **Failure signs**: steps are too long; action labels are unclear.

### `before-after-stack`

- **Input shape**: before/after, wrong/right, problem/solution, old/new.
- **Structure recipe**: two stacked zones or aligned rows with matching fields.
- **Budget rule**: keep contrast fields parallel and short.
- **Failure signs**: contrast is hidden in prose; field counts do not match.

### `myth-fact-qa`

- **Input shape**: repeated myths/facts, questions/answers, misconception/correction.
- **Structure recipe**: compact repeated rows; each row has a label and one clear answer.
- **Budget rule**: correction/result gets stronger visual weight.
- **Failure signs**: answer is buried; too many rows reduce scan speed.

### `data-takeaway-note`

- **Input shape**: one metric, chart, ranking, or trend with interpretation.
- **Structure recipe**: hook, hero metric or compact chart, interpretation sentence, optional 1-2 proof chips.
- **Budget rule**: one main data object is usually enough.
- **Failure signs**: no interpretation; data labels are too dense.

### `mini-framework`

- **Input shape**: one concept with 3-5 facets, rules, or categories.
- **Structure recipe**: central concept plus compact surrounding/stacked facets, or a simple 2x2.
- **Budget rule**: facets stay short and saveable.
- **Failure signs**: framework is not reusable at a glance; facets become long prose.

### `template-note`

- **Input shape**: reusable template, prompt, formula, checklist form, or decision aid.
- **Structure recipe**: hook, template body with labeled slots/rows, bottom usage note.
- **Budget rule**: make the reusable structure visually obvious.
- **Failure signs**: template is buried in explanation; labels are not actionable.
